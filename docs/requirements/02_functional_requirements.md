# 02. 機能要件

各要件にはIDを付与する。優先度: Must（MVP必須）/ Should（早期に欲しい）/ Could（将来検討）。

## 2.1 認証・アカウント管理

| ID | 要件 | 優先度 |
|---|---|---|
| AUTH-01 | 一般ユーザーはメールアドレス＋パスワードで会員登録できる | Must |
| AUTH-02 | 一般ユーザーはログイン・ログアウトできる | Must |
| AUTH-03 | 一般ユーザーはパスワードを再設定できる（メールでのリセットリンク） | Should |
| AUTH-04 | 管理者はメールアドレス＋パスワードでログイン・ログアウトできる | Must |
| AUTH-05 | 管理者アカウントはシステム管理者が発行する（一般ユーザーからの昇格は不可） | Must |
| AUTH-06 | ログインセッションはトークン（JWT等）で管理し、有効期限を設ける | Must |

## 2.2 メニュー管理（管理者）

| ID | 要件 | 優先度 |
|---|---|---|
| MENU-01 | 管理者はメニュー（名称・説明・料金・所要時間）を登録できる | Must |
| MENU-02 | 管理者はメニューを編集・削除できる | Must |
| MENU-03 | 管理者はメニューを公開/非公開に切り替えられる（非公開はユーザー側に表示しない） | Should |
| MENU-04 | メニューにカテゴリ（カット・カラー・パーマ等）を設定できる | Should |
| MENU-05 | メニューごとに対応可能なスタッフを紐付けられる | Should |

## 2.3 スタッフ管理（管理者）

| ID | 要件 | 優先度 |
|---|---|---|
| STAFF-01 | 管理者はスタッフ（氏名・紹介文・写真）を登録・編集・削除できる | Must |
| STAFF-02 | スタッフごとに稼働曜日・稼働時間を設定できる | Must |
| STAFF-03 | スタッフごとに個別休日・臨時休業を設定できる | Should |
| STAFF-04 | スタッフを一時的に非表示（退職等）にできる | Should |

## 2.4 営業設定（管理者）

| ID | 要件 | 優先度 |
|---|---|---|
| BIZ-01 | 管理者は店舗の営業曜日・営業時間を設定できる | Must |
| BIZ-02 | 管理者は休業日（定休日・臨時休業）を設定できる | Must |
| BIZ-03 | 予約枠の時間単位（例: 30分刻み）を設定できる | Must |
| BIZ-04 | 同時に対応可能な予約数の上限（席数相当）を設定できる | Should |

## 2.5 予約（一般ユーザー）

| ID | 要件 | 優先度 |
|---|---|---|
| RSV-01 | ユーザーはメニューを1つ以上選択して予約できる | Must |
| RSV-02 | ユーザーは希望日・時間・（任意で）担当スタッフを指定して空き状況を確認できる | Must |
| RSV-03 | ユーザーは空き枠に対して予約を確定できる | Must |
| RSV-04 | ユーザーは自分の予約一覧（今後・過去）を閲覧できる | Must |
| RSV-05 | ユーザーは予約日時・メニューを変更できる（変更可能期限は要検討） | Should |
| RSV-06 | ユーザーは予約をキャンセルできる（キャンセル可能期限は要検討） | Must |
| RSV-07 | 予約確定・変更・キャンセル時にメール通知を送信する | Should |
| RSV-08 | 予約時に選択したメニューの合計所要時間から必要な枠を自動計算し、重複予約を防止する | Must |

## 2.6 予約管理（管理者）

| ID | 要件 | 優先度 |
|---|---|---|
| ADMRSV-01 | 管理者は全予約を日別・スタッフ別のカレンダー/一覧で閲覧できる | Must |
| ADMRSV-02 | 管理者は電話等で受けた予約を代理登録できる | Must |
| ADMRSV-03 | 管理者は予約内容の変更・キャンセルができる | Must |
| ADMRSV-04 | 管理者は予約を検索（顧客名・日付・スタッフ・メニューで絞り込み）できる | Should |
| ADMRSV-05 | 管理者は予約ステータス（予約済み/来店済み/キャンセル/無断キャンセル）を更新できる | Should |

## 2.7 顧客管理（管理者）

| ID | 要件 | 優先度 |
|---|---|---|
| CUST-01 | 管理者は顧客一覧を閲覧・検索できる | Must |
| CUST-02 | 管理者は顧客詳細（プロフィール・予約履歴）を閲覧できる | Must |
| CUST-03 | 管理者は顧客ごとにメモ（施術履歴・注意事項等）を登録できる | Should |

## 2.8 通知

| ID | 要件 | 優先度 |
|---|---|---|
| NOTI-01 | 予約確定時にユーザーへ確認メールを送る | Should |
| NOTI-02 | 予約前日等にリマインドメールを送る | Could |
| NOTI-03 | 管理者へ新規予約・キャンセル発生時に通知する | Could |

## 2.9 ダッシュボード（管理者）

| ID | 要件 | 優先度 |
|---|---|---|
| DASH-01 | 管理者は当日の予約件数・スケジュールを一目で確認できる | Should |
| DASH-02 | 管理者は週間・月間の予約状況をカレンダー表示で確認できる | Should |
