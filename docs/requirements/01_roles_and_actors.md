# 01. 利用者・ロール定義

## 1.1 ロール一覧

| ロール | 概要 | 認証 |
|---|---|---|
| 一般ユーザー（顧客） | メニューを選び、日時・担当スタッフを指定して予約する | 必要（会員登録・ログイン） |
| 管理者 | 予約全体・顧客・スタッフ・メニュー・営業設定を管理する | 必要（管理者ログイン） |
| 未ログインユーザー | 会員登録前の閲覧者 | 不要（閲覧のみ、予約にはログインが必要） |

スタッフは個別アカウントを持たず、管理者が管理するマスタデータ（担当スタッフ選択肢）として扱う。

## 1.2 一般ユーザーができること
- 会員登録・ログイン・ログアウト・パスワード再設定
- 自分のプロフィール（氏名・電話番号・メール等）の閲覧・編集
- メニュー一覧・料金・所要時間の閲覧
- 空き状況（日付・時間帯・担当スタッフ別）の閲覧
- 予約の新規作成・内容変更・キャンセル
- 自分の予約履歴・今後の予約一覧の閲覧
- 予約確定・変更・キャンセル時の通知（メール等）受信

## 1.3 管理者ができること
- 管理者ログイン・ログアウト
- 予約の一覧閲覧・検索・新規作成（電話予約の代理登録）・変更・キャンセル
- 顧客情報の一覧・詳細閲覧、検索、メモ登録
- メニュー（施術内容・料金・所要時間）の登録・編集・削除・公開/非公開切り替え
- スタッフ情報（氏名・対応可能メニュー・稼働曜日/時間）の登録・編集・削除
- 営業日・営業時間・休業日・予約枠（時間粒度）の設定
- 予約状況のカレンダー/一覧表示（日別・スタッフ別）
- ダッシュボードでの当日/週間予約状況の把握

## 1.4 権限に関する原則
- 一般ユーザーは自分が作成した予約のみ閲覧・編集・キャンセル可能。他ユーザーの予約や顧客情報は閲覧不可。
- 管理者は全顧客・全予約・マスタデータに対してフル権限を持つ。
- 管理者権限に段階（オーナー/スタッフ管理者など）を設けるかは [[06_open_questions]] を参照。
