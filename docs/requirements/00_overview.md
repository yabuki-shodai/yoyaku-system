# 00. 概要・背景

## 1.1 プロジェクト名
yoyaku-system（美容院予約管理システム）

## 1.2 背景・目的
美容院（単一店舗を想定）における予約受付・管理業務を電子化する。
- 顧客: 電話や来店予約に頼らず、Web上でいつでも空き状況を確認し予約できるようにする。
- 店舗（管理者）: 予約状況・顧客情報・スタッフの稼働状況を一元管理し、予約受付や台帳管理の手間を削減する。

## 1.3 対象範囲（スコープ）
- 単一店舗での運用を前提とする（複数店舗・フランチャイズ対応は本フェーズのスコープ外）。
- 利用者ロールは「一般ユーザー（顧客）」と「管理者」の2種類のみとする。スタッフ個別ログインは持たず、スタッフ情報は管理者が管理するマスタデータとして扱う。
- 決済機能（オンライン決済）は本フェーズのスコープ外とし、予約と来店・会計は別管理とする。

## 1.4 システム構成
- `apps/user`: 一般ユーザー向け予約Webアプリ（Next.js）
- `apps/admin`: 管理者向け管理画面（Next.js）
- `apps/api`: バックエンドAPI（NestJS）
- `packages/shared`, `packages/ui`: フロントエンド間で共有するロジック・UIコンポーネント
